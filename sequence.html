4.30 Resource Sequence - Content

Clinical Genomics Work GroupMaturity Level: 0Compartments: Patient

This resource records information about sequence of a moleculde. E.g. a DNA sequence.

4.30.1 Scope and Usage

This resource covers data about the sequence of a macromolecule, including:

  • Type of molecule (DNA, RNA, or protein)
  • Bases of the sequence
  • How does the sequence align to a reference genome
  • Tissue sample from which these molecules are extracted
  • Quantity of these molecules present in the sample

4.30.2.2 Constraints

  • tla-1: Start and end position should be non-negative, and end position must be larger than start position. (xpath: boolean(f:start/@value >= 0) and boolean(f:start/@value >=0) and boolean(f:end/@value > f:start/@value))
  • tla-2: CIGAR string should conform with extended-cigar's syntax. (xpath: matches(f:cigar/@value, '\*|([0-9]+[MIDNSHPX=])+'))
  • Start position is inclusive and end position of exclusive. E.g. a start position `123` and end position of `124` means a single base located at base `123` (0-based).
  • The type of chromosome is string (instead of code or Coding) because sequence represented by the resource can be one from other species, and there are currently no ontology that handles such a range of chromosome identifiers. For similar reason, type of genomeAssembly is also string.
  • The field `quantity` records the amound of molecules encoded by the sequence present in the sample. A common use case is using this to represent gene expression in the case of `RNA` sequences, by recording the amound of certain `RNA` sequences present in the sample.
  • The value of `referenceBases` SHOULD match the bases in reference genome (i.e. the value of `genomeBuild`) at genomic position specified.
